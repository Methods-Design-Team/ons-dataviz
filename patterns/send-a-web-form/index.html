    When to use this pattern

    Use this pattern when:

    • a user needs to get in touch with us but they do not need an instant response or to talk to someone in the contact centre directly
    • we can give a specific response time, such as “We will respond within 48 hours”

    When not to use this pattern

    Do not use this pattern if the user needs to:

    • speak to someone at the contact centre, for example, if the details about a business need to change
    • fix an urgent issue that needs a fast response, like problems with the service

    How to use this pattern

    Give users a set of categories to choose from on the web form. These should be based on the most common user needs identified through user research. This helps contact centre staff deal with enquiries.

    Only ask the user for the contact details the service needs to handle the request.

    The following could be included on a web form:

    • What do you need help with? (radio buttons)
    • Please provide more detail (textarea)
    • Name (input)
    • Email address (input)
    • Send message (button)
    • Description of what we do with personal data and a link to the privacy statement

    How to check a web form

    To help users fill in a web form, you should:

    • check they have selected a radio button
    • allow them to paste into the fields
    • check they have entered something in all of the fields
    • show an error message if they have not selected a required radio button or any of the fields are empty

    Error messages

    Use the correct errors pattern and show the error details above the relevant field or radios.

    Guidance on writing error messages can be found in the radios component, textarea component or input component.
